# Web 管理后台操作手册

> **版本**: v1.0 (T24)
> **适用角色**: super_admin / ops / sales / compliance
> **系统入口**: `/admin/login`
> **核心模块**: 全链路 6 阶段看板 · 商机管理 · 采集任务 · 合规审核 · 批量操作 · 数据导出

---

## 1. 系统概述

### 1.1 架构与角色体系

管理后台基于 **4 级角色权限模型**，每个角色拥有独立的可访问页面与操作能力：

| 角色 | 名称 | 主要职责 | 权限等级 |
|------|------|---------|---------|
| **super_admin** | 超级管理员 | 账号管理、权限配置、系统健康检查、明文数据导出 | 最高 |
| **ops** | 运营 | 采集任务管理、异常数据处理、数据分析、批量操作、脱敏数据导出 | 中高 |
| **compliance** | 合规专员 | 任务合规审核、违规数据复核、审计日志查看 | 中等 |
| **sales** | 销售 | 商机查看、商机分级、手动触达、跟进记录、状态管理 | 标准 |

### 1.2 登录与登出

**登录**: 访问 `/admin/login`，输入用户名和密码
- 首次登录后由 super_admin 分配角色
- 登录会话有效期由系统配置

**登出**: 点击页面右上角「退出登录」

**常见问题**:
- 登录失败: 检查用户名/密码，或联系管理员确认账号状态
- 页面无权限: 确认当前角色是否拥有该页面权限

### 1.3 页面导航说明

顶部导航栏按模块划分：
- **首页** — 全局关键指标概览
- **采集阶段** — 爬虫任务与原始数据管理
- **清洗结构化** — 数据清洗与结构化结果
- **合规校验** — 合规规则与违规数据管理
- **商机分级** — 商机评分与标签管理
- **客户触达** — 消息发送与触达结果
- **销售闭环** — 商机分配、跟进与成交
- **异常数据池** — 异常数据集中处理
- **分渠道漏斗** — 渠道效果分析
- **批量操作中心** — 异步批量任务管理
- **数据导出中心** — 数据导出管理

## 3. 运营操作手册（ops 角色）

### 3.1 采集任务管理

**操作路径**: 采集阶段 → 任务列表 → [创建任务] / [启动] / [暂停] / [重试]

**创建采集任务**:
1. 点击「创建任务」按钮
2. 选择采集渠道（7 大渠道可选）
3. 填写任务参数（关键词、目标范围、速度级别）
4. 勾选合规协议：
   - 确认数据用途仅用于业务分析
   - 确认不采集隐私敏感数据
   - 确认留存周期符合合规要求
5. 提交任务
   - 若渠道开启审批 → 任务进入「待审核」状态，需 compliance 审核通过
   - 若渠道免审 → 任务直接进入「待启动」状态

**任务启停操作**:
- **启动**: 选中任务 → 点击「启动」→ 任务开始执行
- **暂停**: 运行中任务 → 点击「暂停」→ 任务挂起，可恢复
- **终止**: 运行中任务 → 点击「终止」→ 任务结束，不可恢复
- **重试**: 已失败任务 → 点击「重试」→ 从失败断点续爬

**批量任务操作**:
- 进入「批量操作中心」→ 选择「批量启动」/「批量暂停」/「批量删除」
- 填写 item_ids（逗号分隔的任务 ID 列表）
- 提交后系统返回 batch_id，可查看进度条

### 3.2 全链路看板与数据监控

**操作路径**: 首页 → 全链路看板

**6 阶段数据查看**:
- **采集阶段**: 查看各渠道采集量、原始数据明细
- **清洗结构化**: 查看结构化字段提取结果、企业信息识别
- **合规校验**: 查看合规检查结果、高风险标记
- **商机分级**: 查看商机评分、行业标签、意向等级
- **客户触达**: 查看触达渠道、发送状态、成功率
- **销售闭环**: 查看分配、跟进、成交状态

**筛选与排序**:
- 按时间范围筛选（今日 / 本周 / 本月 / 自定义）
- 按渠道类型筛选（7 大渠道）
- 按状态筛选（正常 / 异常 / 待处理）
- 表格支持按列排序（点击表头）

**操作**:
- 点击数据行进入「单商机全生命周期追踪」页（时间线视图）
- 点击「导出」按钮导出当前筛选条件下的数据

### 3.3 异常数据池处理

**操作路径**: 异常数据池

**异常类型分类**:
| 类型 | 说明 | 处理方式 |
|------|------|---------|
| cleaning_failed | 数据清洗失败 | 重新入库 / 废弃 |
| compliance_blocked | 合规拦截 | 强制放行 / 永久驳回 |
| extract_error | 关键字段提取失败 | 手动修正 / 重新清洗 |
| duplicate_suspect | 疑似重复数据 | 合并 / 标记正常 |
| risk_blocked | 风控规则拦截 | 复核放行 / 永久驳回 |

**单条处理**:
1. 在异常列表中找到目标数据
2. 点击对应操作按钮：
   - **重新入库**: 数据重新进入处理流水线
   - **废弃**: 标记为不可用数据
   - **标记误判**: 异常标记移除，数据恢复正常

**批量处理**:
- 「批量操作中心」→ 选择操作类型
- 填写异常 item_ids，选择批量动作
- 提交后异步执行，实时查看进度

**统计分析**:
- 异常总量与今日新增
- 各类型占比（饼图/条形图）
- 各渠道异常率趋势
- 7 日异常数量变化曲线

### 3.4 分渠道转化效果分析

**操作路径**: 分渠道漏斗看板

**核心指标**（每个渠道独立计算）:
- **采集量**: 渠道原始采集条数
- **有效线索**: 经过清洗校验通过数
- **商机数**: 被标记为商机的数量
- **触达数**: 已发起触达的商机数
- **跟进数**: 有销售跟进记录的数量
- **成交数**: 最终标记成交的数量
- **转化率**: 各阶段转化率与全链路转化率
- **单商机成本**: 渠道投入 / 商机数
- **成交周期**: 从采集到成交的平均天数

**排行榜**:
- 转化率 Top 3 渠道
- 成交数 Top 3 渠道
- 单商机成本最低的 3 个渠道

**周期对比**:
- 选择「按周」或「按月」查看
- 对比不同时间周期的渠道效果变化

### 3.5 数据导出与下载

**操作路径**: 数据导出中心

**导出操作**:
1. 选择导出阶段（采集 / 清洗 / 商机 / 异常 / 全量）
2. 可选：筛选条件（时间范围、渠道、状态）
3. **重要**: 确认「明文导出」选项
   - **默认未勾选**: 导出脱敏数据（无联系方式等隐私字段）
   - **super_admin 可选**: 勾选后导出原始数据（需额外权限）
4. 填写导出原因（审计留痕）
5. 提交导出任务

**下载方式**:
- 大数据量（>1000 条）: 异步生成 → 完成后通知下载 → 点击下载按钮
- 小数据量: 直接生成 → 立即下载
- 文件格式: CSV（UTF-8 编码）

**注意事项**:
- 所有导出操作均在审计日志中留痕，包含操作人、时间、导出内容范围
- 明文导出仅限 super_admin，且每次操作记录理由
- 导出文件有时效期，过期后需重新申请

## 4. 销售操作手册（sales 角色）

### 4.1 商机看板与跟进

**操作路径**: 销售闭环 → 商机列表

**查看商机**:
- 按「我的商机」/「全量商机」切换视图
- 筛选条件：状态（待跟进 / 跟进中 / 成交 / 流失 / 无效）、意向等级、渠道来源
- 排序：按商机评分、创建时间、最近跟进时间排序

**查看详情**:
点击商机 ID 进入「全生命周期时间线」:
- **采集来源**: 原始渠道与采集时间
- **清洗结果**: 识别出的企业信息、关键字段
- **合规状态**: 合规检查结果与风险等级
- **评分与标签**: 自动评分 + 人工标签
- **触达记录**: 已发送的消息与状态
- **分配记录**: 销售人员分配历史
- **跟进记录**: 销售跟进日志（时间 + 内容）
- **最终状态**: 成交/流失/无效，含结束时间与备注

### 4.2 商机分级与调级

**操作路径**: 商机分级 → 操作列

**手动调整商机等级**:
- 等级选项：高意向 / 普通 / 低意向 / 垃圾
- 操作：选择目标商机 → 点击「调级」→ 选择新等级 → 确认

**修改商机打分**:
- 点击「编辑评分」→ 调整 0-100 分值
- 填写调整理由（自动记录审计日志）

**补充行业/地域标签**:
- 点击「编辑标签」→ 添加/移除标签
- 系统内置行业标签库 + 自定义标签

**加入黑名单**:
- 对无效/骚扰商机 → 点击「加入黑名单」
- 黑名单商机不再进入后续处理流程
- 操作需二次确认，避免误操作

### 4.3 手动发起触达

**操作路径**: 客户触达 → 选择商机 → 发起触达

**触达渠道选择**:
- 短信: 需确保手机号字段非空
- 邮件: 需确保邮箱字段非空
- 企微消息: 需确保企业微信 ID 绑定

**发送流程**:
1. 选择目标商机
2. 选择触达渠道
3. 填写/选择消息内容（支持模板）
4. 二次确认发送
5. 系统返回发送状态（成功 / 失败 / 发送中）

**失败重试**:
- 查看触达记录 → 找到失败条目 → 点击「重试」
- 或更换触达渠道重新发送

### 4.4 录入跟进记录

**操作路径**: 销售闭环 → 商机详情 → 「新增跟进」

**跟进内容**:
- 跟进日期（默认今日，可修改）
- 跟进方式（电话 / 邮件 / 面谈 / 线上会议）
- 客户反馈内容
- 下一步计划
- 附加标签（客户态度、成交概率等）

**跟进状态流转**:
- 商机状态自动根据跟进情况更新（跟进中 → 高意向 → 成交）
- 可手动覆盖状态（需确认）

### 4.5 关闭商机

**操作路径**: 商机详情 → 「关闭商机」

**关闭类型**:
- **成交**: 标记成交金额、成交时间、备注
- **流失**: 标记流失原因（价格过高、无预算、需求不符、竞品选择、失联等）
- **无效**: 标记无效原因（空号、错号、非目标客户、重复商机等）

**注意**:
- 商机关闭后不可恢复
- 关闭操作永久留痕，包含操作人、时间、原因
- 建议每周统一清理已判断结果的商机

## 5. 合规操作手册（compliance 角色）

### 5.1 采集任务合规审核

**操作路径**: 合规审核 → 待审核任务

**审核流程**:
1. 查看待审核列表，按渠道/提交时间筛选
2. 点击查看任务详情：采集范围、参数、合规协议勾选状态
3. 执行审核操作:

**【通过】**:
- 任务状态更新为「待启动」
- 通知任务提交人（站内消息）
- 任务可由 ops 启动执行

**【驳回】**:
- 填写驳回原因（必填，>10 字）
- 任务状态更新为「已驳回」
- 通知提交人 → 可修改参数后重新提交审核
- 驳回原因永久留痕，不可编辑

### 5.2 审核规则配置

**操作路径**: 合规设置 → 审核规则（仅 super_admin / compliance 可见）

**可配置项**:
- **渠道是否需要审批**: 对 7 大渠道分别开启/关闭自动审批
- **默认审批角色**: 指定某类任务的审批人角色
- **合规协议文本**: 自定义任务提交时显示的协议内容
- **数据留存周期**: 定义各渠道数据默认保留时长
- **违禁关键词**: 添加/移除违规采集关键词，命中自动拦截

### 5.3 违规数据复核与处理

**操作路径**: 异常数据池 → 合规拦截 / 风控拦截类型

**复核操作**:
- **强制放行**: 确认数据合规，允许进入后续流程（需填写理由）
- **标记误判**: 合规规则误命中，修正规则阈值（如有）
- **永久驳回**: 确认数据违规，彻底屏蔽，不再进入流水线

**合规等级调整**:
- 调整单条数据的合规等级（低/中/高风险）
- 调整脱敏规则：选择更严格或更宽松的脱敏方案

### 5.4 合规审计日志查看

**操作路径**: 审计日志 → 按操作类型筛选

**可查询的合规相关操作**:
- 审核通过 / 驳回记录
- 违规数据复核记录
- 合规规则变更
- 明文数据查看记录
- 明文导出记录

**查询条件**:
- 时间范围
- 操作人
- 操作类型
- 对象 ID

**审计日志特点**:
- 不可修改、不可删除
- 记录操作人、时间、操作前后快照
- 支持导出（仅限 super_admin / compliance）

## 6. 管理员操作手册（super_admin 角色）

**创建账号**:
1. 点击「新建账号」
2. 填写用户名（唯一）、密码（>8 位，含数字+字母）、邮箱

4. 确认创建（需 super_admin 权限）

**账号状态管理**:
- 启用 / 禁用账号
- 重置密码（发送临时密码到邮箱）
- 修改角色（注意：角色变更可能影响可见页面）

## 8. 操作规范与合规要求

### 8.1 数据使用承诺

所有系统用户需在首次登录时确认以下承诺:
- 数据仅用于合法业务用途
- 不得将系统内数据导出用于个人用途
- 不得向第三方泄露（除非取得法律授权）
- 遵守数据最小化原则，只查看/操作职责范围内数据

### 8.2 隐私字段查看规范

**默认行为**:
- 所有包含手机号、邮箱、联系人姓名的字段，在页面上默认显示为脱敏格式（`138****5678`、`j***@example.com`）

**查看明文**:
1. 点击字段旁的「👁」图标
2. 二次确认（弹窗提示）
3. 显示明文（当前页面仅本次会话可见）
4. 操作自动写入审计日志（含查看人、查看时间、查看对象）

**权限限制**:
- 仅 super_admin 可查看明文
- ops / compliance / sales 角色始终看到脱敏内容

### 8.3 批量操作风险提示

批量操作与单条操作不同:
- **不可撤回**: 批量操作执行后，逐条撤销需人工逐条恢复
- **影响面大**: 错误选择 item_ids 可能影响大量数据
- **审核严格**: 高危批量操作需 super_admin 权限 + 操作理由留痕

**最佳实践**:
1. 小批量测试（1-3 条）→ 确认操作效果正常
2. 明确筛选条件，避免 item_ids 填错
3. 在业务低峰期执行大规模批量操作

### 8.4 审计日志不可删除说明

系统所有操作均在审计日志中留痕，特点:
- 一旦记录，不可修改、不可删除
- 包含操作前后数据快照，可还原变更
- 保留期: ≥ 6 个月
- 支持导出（合规审核）

## 9. 常见问题 FAQ

### 9.1 登录与权限类

**Q: 登录后看不到某些页面？**
A: 页面可见性由角色决定。请确认您的角色是否拥有对应权限，或联系 super_admin 查看。

**Q: 页面显示「权限不足」？**
A: 您尝试执行的操作需要更高权限。联系管理员确认是否需要提升角色。

**Q: 登录会话很快过期？**
A: 默认会话有效期由系统配置决定。长时间不操作会自动登出，这是安全策略。

### 9.2 数据查询与展示类

**Q: 列表页数据为空？**
A: 可能（1）当前筛选条件下确实无数据，尝试放宽筛选；（2）Redis 服务不可用导致模拟数据展示异常，联系管理员。

**Q: 渠道漏斗数据显示异常（全是 0）？**
A: 该渠道尚无采集任务运行或采集时间过短。请先创建并启动该渠道的采集任务。

**Q: 商机时间线数据不全？**
A: 某些阶段可能没有操作记录（例如尚未发起触达），这是正常的。时间线只显示已执行的节点。

### 9.3 异常处理类

**Q: 页面显示「请求失败，请重试」？**
A: 网络或后端服务异常。尝试刷新页面，或稍后重试。如持续发生，请联系管理员并记录出错页面 URL。

**Q: 批量操作进度条长时间停滞？**
A: 可能（1）任务量过大，需耐心等待；（2）任务执行异常，查看进度旁的错误信息；（3）Redis 服务不可用，任务降级为同步执行。

**Q: 导出文件下载失败？**
A: 导出文件有有效期（通常 30 分钟），过期后需重新提交导出申请。也可能是浏览器拦截，请检查下载提示。

### 9.4 批量操作类

**Q: 提交批量任务后页面显示「items empty」？**
A: item_ids 未正确填写或格式错误。请确保使用英文逗号分隔 ID，无多余空格或换行。

**Q: 批量任务提交后没有反应？**
A: 检查是否获取了 batch_id。如果点击后无响应，可能是前端 JS 加载异常。刷新页面后重试。

### 9.5 导出与下载类

**Q: 导出的 CSV 中文显示乱码？**
A: 文件使用 UTF-8 编码。请使用支持 UTF-8 的工具打开（Excel 需选择「文件 → 打开 → 文本文件 → UTF-8」，或使用 LibreOffice、VS Code 等）。

**Q: 导出文件字段太多？**
A: 当前导出包含该阶段所有字段。如需精简字段，请在筛选条件中指定需要的列（部分导出支持字段选择）。

**Q: 手机号/邮箱全是星号？**
A: 系统默认脱敏导出。如需明文导出，请由 super_admin 操作，并确保用途合法、合规协议已签署。

## 11. 操作自检清单

每次进行关键操作时，建议对照以下清单:

**创建采集任务** □ 选择了正确渠道 □ 参数合理 □ 已勾选合规协议 □ 若需审批，已通知审核人

**发起批量操作** □ item_ids 填写正确（建议复制而非手动输入） □ 先小批量测试效果 □ 已填写操作原因

**导出数据** □ 确认是否需要明文（默认应使用脱敏） □ 导出用途符合数据使用承诺 □ 文件下载后妥善保管

**关闭商机** □ 确认最终状态（成交/流失/无效） □ 已填写状态原因 □ 确认操作后不可恢复

**查看明文** □ 确为业务必要（非好奇查看） □ 不复制明文到外部系统 □ 不截图标注明文字段
