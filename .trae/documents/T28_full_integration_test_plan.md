## 一、现状调研结论

### 1.1 模块结构

```
core/spider_core/                    ← T25 底层引擎
  ├── page_renderer.py              页面智能渲染
  ├── smart_analyzer.py             列表/链接/附件智能识别
  ├── field_extractor.py            CSS/XPath/正则字段提取
  ├── pdf_parser.py / image_parser.py 附件内容解析(OCR)
  ├── attachment_parser.py          统一附件入口
  ├── dedup_store.py                增量去重
  ├── alert_manager.py              告警管理
  ├── rule_engine.py                规则执行器
  └── sdk.py                        对外 SDK

business/custom_spider/              ← T26 业务层
  ├── data_models.py                数据表模型(4张)
  ├── pydantic_models.py            请求/响应DTO
  ├── repository.py                 数据访问层
  └── service.py                    方案 CRUD / 测试运行 / 定时调度

web_admin/                          ← T27 交互层
  ├── api/crawl_config.py           方案管理/预览/监控 API
  ├── pages.py                      4个页面路由
  ├── static/js/crawl_plans.js      方案管理列表
  ├── static/js/crawl_editor.js     五步可视化配置编辑器
  ├── static/js/crawl_monitor.js    任务监控仪表
  └── static/js/crawl_fields.js     字段模板库

对接依赖：
  ├── T03 调度能力(cron/手动触发)
  ├── T04 数据库(原始爬虫表扩展)
  ├── T06 合规校验(robots + 敏感内容)
  ├── T10 清洗流水线(数据标准化/商机生成)
  └── 角色权限体系(super_admin / ops / sales / viewer)
```

### 1.2 已知待验证问题

1. **建表链路**：`data_models.create_tables()` 是否已被启动流程调用
2. **Redis 去重**：`DedupStore` 使用 Redis，需验证连接与键空间管理
3. **T25 引擎调用**：`service._call_t25_engine()` 需验证实际采集能力
4. **权限装饰器**：`_render_with_permission` 需验证角色分级可见性
5. **页面消毒**：`_sanitize_html` / `_mask_sensitive_text` 需验证实际效果
6. **定时任务**：方案启停 → APScheduler/调度中心 的实际联动

### 缺陷分级标准

| 级别 | 定义 | 响应要求 |
|------|------|---------|
| **P0 阻塞** | 主流程无法跑通、数据丢失、页面崩溃、系统不可用 | 当日必须修复 |
| **P1 严重** | 核心功能异常、但有绕过方案；数据一致性问题；权限失效 | 2 个工作日内修复 |
| **P2 一般** | 交互体验问题、提示文案不清晰、页面样式偏差 | 本轮测试周期内修复 |

## 六、风险与应对

| 风险 | 描述 | 应对方案 |
|------|------|---------|
| **外部站点不可达** | 采集目标站点可能反爬、限频、页面结构变化 | 准备 2-3 个备选测试站点；离线页面快照作为 fallback |
| **OCR 依赖未安装** | 图片/PDF OCR 依赖 tesseract/poppler 等第三方库 | 验证 `.env` 中开关逻辑，未安装时 graceful fallback，不阻塞其他测试 |
| **定时任务调度耦合** | T26 的方案启停是否真的能调用到调度中心 | 设计 mock 方案：可用 APScheduler 本地实例验证，也可手动验证 |
| **权限装饰器遗漏** | 部分 API 可能未正确包裹权限校验 | 走查 `web_admin/api/crawl_config.py` 所有路由装饰器 |
| **Redis 环境差异** | 开发/测试环境 Redis 配置不一致导致去重异常 | 在测试启动脚本中集中初始化 Redis，统一键前缀 |
| **与现有爬虫冲突** | 定制采集与 T18/T19 自动化爬虫可能使用相同的代理池/UA | 确保采集任务有独立的优先级/限流参数，互不干扰 |
